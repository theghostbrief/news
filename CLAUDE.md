# News Digest Pipeline

## Описание проекта
Сбор новостей с Perplexity через Chrome-расширение (папка `extension/`), обработка в авторские Facebook-дайджесты на русском языке.

## Пути файлов
- **Входные файлы**: `./input_*.json` — JSON-файлы от расширения
- **Выходные файлы**: `./output/` — готовые дайджесты (`digest_YYYY-MM-DD_partN.txt`)
- **Стиль комментариев**: `./prompt.md`
- **Формат сборки**: `./assembly_prompt.md`
- **Настройки (курс, хэштеги, граница)**: `./config.md`

## Процесс обработки
1. Пользователь загружает `input_*.json`
2. Проверить качество контента (нет ли мусора из сайдбара)
3. Разбить на части по указанию пользователя (обычно 13-17 статей на часть)
4. Для каждой части сгенерировать дайджест по правилам из `prompt.md` и `assembly_prompt.md`
5. Сохранить результат в `./output/`

## Формат дайджеста
- `#новости` в начале
- Пронумерованные авторские комментарии + ссылка на оригинал
- Упоминание курса (из `config.md`) — в середине списка, как случайная ремарка
- Граница/дисклеймер + хэштеги (из `config.md`) — в конце


### Делегирование задач субагентам

При получении задачи от пользователя — сначала оценить:
- **Сам:** быстрые правки (< 2 мин), обсуждения, анализ, вопросы, мелкие фиксы
- **Субагент:** код > 50 строк, новые модули, рефакторинг, UI-изменения, исследования, любые задачи > 5 мин

При делегировании субагенту:
- Составить детальное ТЗ с контекстом, файлами для чтения, ожидаемым результатом
- Запустить в фоне
- Сообщить пользователю что запущено
- По завершении — дать краткий отчёт о результате
- Можно запускать несколько субагентов параллельно на независимые задачи

Цель: максимальная параллельность и минимальное время ожидания пользователя.

## Автономность

Агент полностью автономен в рамках этого проекта. Все решения по коду, архитектуре, тестированию, безопасности и качеству принимаются самостоятельно без запроса подтверждения у пользователя.

**Привлекать пользователя только когда:**
- Нужны креды или доступ к внешним сервисам (VPS, API-ключи, токены)
- Подключение к удалённому серверу
- Финальная демонстрация результата
- Неоднозначные продуктовые решения (не технические)
